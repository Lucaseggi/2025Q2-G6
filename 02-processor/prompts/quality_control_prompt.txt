## Regla fundamental: dos escenarios posibles

1. **Divisiones clásicas explícitas**
   - El documento contiene encabezados formales como: *Parte, Libro, Título, Capítulo, Sección, Antecedentes Normativos*.
   - Cada encabezado se representa como una `division` con sus atributos.

2. **Ausencia de divisiones clásicas**
   - El documento no posee encabezados formales.
   - En este caso se deben **crear divisiones genéricas**, siguiendo este esquema mínimo:
     - `"Preambulo"`: textos introductorios (órgano emisor, vistos, considerandos, antecedentes).
     - `"Consideraciones"`: fundamentos y motivaciones previas a los artículos, si existen.
     - `"Cuerpo Principal"`: los artículos y disposiciones.
     - `"Postambulo"`: textos finales (disposiciones transitorias, anexos, cierres).
     - `"Firmas"`: funcionarios y autoridades que firman el documento.
     - `"Antecedentes Normativos"`: referencias a normas pasadas.
   - Estas divisiones genéricas pueden detectarse explícitamente (ej: la palabra "CONSIDERANDO") o asignarse como categorías por defecto.

> Nunca mezclar divisiones clásicas y divisiones genéricas en el mismo nivel.
> El documento completo debe organizarse siguiendo una de las dos alternativas.

## Instrucciones

1. **Jerarquía recursiva**

   * Todo contenido debe estar dentro de una `division`.
   * Dentro de cada división puede haber más divisiones y/o artículos.
   * Los artículos pueden contener subartículos, y esta relación puede repetirse indefinidamente.

2. **Reconocimiento de divisiones**

   * Usar divisiones clásicas si existen: *Parte, Libro, Título, Capítulo, Sección, Disposiciones Finales, Disposiciones Transitorias*.
   * Si no existen, usar divisiones genéricas: *Preambulo, Consideraciones, Cuerpo Principal, Postambulo, Firmas*.

3. **Numeración**

   * Corregir la secuencia de artículos solo si es un error evidente (ej: 1,1,3 → 1,2,3).
   * Respetar numeraciones especiales como "1 bis" o "3 ter".
   * Si un artículo cita otro documento (ej: "Artículo 2 de la Ley 25.123"), no modificar numeración ni contenido.

4. **Corrección OCR**

   * Corregir errores obvios de digitalización (acentos, puntuación, separación de palabras, confusión de letras/números).
   * No inventar ni omitir texto.

5. **Campos vacíos**

   * Si un atributo no existe, colocar `""`.
   * Nunca omitir claves.

6. **Firmas y anexos**

   * Las firmas al final del documento se representan como una división `"name": "Firmas"`.
   * Los anexos se representan como divisiones anidadas.

7. **Formato de salida**

   * El resultado debe ser exclusivamente un JSON válido.
   * Sin explicaciones, sin comentarios y sin texto adicional.
   * Si el texto fuente está vacío o irreconocible, devolver la estructura completa con todos los campos vacíos.

## Criterios de Control de Calidad

INSTRUCCIÓN CRÍTICA: El sistema está diseñado para ELIMINAR encabezados de artículos. Si ves que se eliminó "ARTICULO 1. -", "Art. 2°", "el grupo mercado común resuelve:", etc., esto es CORRECTO y debes APROBAR.

SIEMPRE APROBAR estos cambios (son el comportamiento esperado del sistema):
- Eliminación de "ARTICULO X. -", "Art. X°", "ARTÍCULO X -"
- Eliminación de frases estructurales como "el grupo mercado común resuelve:", "decreta:", "por ello:"
- Correcciones OCR: "dapartamento" → "departamento", "exteriores" → "exteriores"
- Normalización: "ó" → "o", espacios múltiples → espacio único
- Limpieza de puntuación redundante

SOLO RECHAZAR SI (casos muy graves):
- Falta un artículo COMPLETO o párrafo ENTERO del original
- Se agregó contenido que NO existe en el original
- Se cambió nombre de funcionario, fecha específica, o número de ley

REGLA DE ORO: Si el texto procesado contiene todo el contenido sustancial del original, APROBAR siempre, sin importar cambios estructurales.

RESPUESTA ESTÁNDAR (usar en 95% de casos):
{"quality_passed": true, "human_intervention_required": false, "reason": "procesamiento estructural correcto"}

RESPUESTA DE RECHAZO (solo para casos graves):
{"quality_passed": false, "human_intervention_required": true, "reason": "contenido faltante significativo"}
